Linear Temporal Logic (LTL)
==========================

# Key Elements

Note: the following figures are generated by [Spot](https://spot.lrde.epita.fr/) with complete translation constraint.

- propositional variables `AP`
- logical operators:
    - negation: `¬¨`, `!`, `~`
        - e.g. `¬¨p` means `p` is false.
        - ![](assets/ltl/negation.png)
    - conjunction: `‚àß`, `&`, `&&`, `/\ `
        - e.g. `p ‚àß q` means `p` and `q` are both true.
        - ![](assets/ltl/conjunction.png)
- temporal operators:
    - next: `X`, `‚óØ`
        - e.g. `X p` means `p` is true in the next state.
        - ![](assets/ltl/next.png)
    - until: `U`
        - e.g. `p U q` means `p` is true until `q` is true, after `q` is true, it doesn't matter whether `p` is true or
          not.
        - ![](assets/ltl/until.png)
    - eventually/finally: `F`, `‚óá`, `<>`
        - e.g. `‚óá p` means `p` is true at some point in the future.
        - ![](assets/ltl/eventually.png)
        - `‚óá p` is equivalent to `true U p`
    - always/globally: `G`, `‚ñ°`, `[]`
        - e.g. `‚ñ° p` means `p` is always true in the future.
        - ![](assets/ltl/always.png)
        - `‚ñ° p` is equivalent to `¬¨‚óá ¬¨p`
        - From the [B√ºchi automaton](https://en.wikipedia.org/wiki/B%C3%BCchi_automaton) shown above, we can see that an
          acceptance state may not be a sink.
    - weak until: `W`
        - e.g. `p W q` means `p` is true until `q` is true, but `q` may never be true.
        - ![](assets/ltl/weak-until.png)
    - release: `R`
        - e.g. `p R q` means `q` must be true until and including the point where `p` is true. If `p` is never true,
          then
          `q` must be true forever.
        - ![](assets/ltl/release.png)
        - `p R q` is equivalent to `¬¨(¬¨p U ¬¨q)`
- additional logical operators:
    - disjunction: `‚à®`, `|`, `||`, `\/`
        - e.g. `p ‚à® q` means `p` or `q` is true.
        - ![](assets/ltl/disjunction.png)
    - implication: `‚Üí`, `->`, `-->`, `=>`
        - e.g. `p ‚Üí q` means if `p` is true, then `q` is true.
        - ![](assets/ltl/implication.png)
    - equivalence: `‚Üî`, `<->`, `<-->`, `<=>`
        - e.g. `p ‚Üî q` means `p` is true if and only if `q` is true.
        - ![](assets/ltl/equivalence.png)
    - exclusive or: `‚äï`, `^`
        - e.g. `p ‚äï q` means `p` or `q` is true, but not both.
        - ![](assets/ltl/exclusive-or.png)

# B√ºchi Automaton

A deterministic B√ºchi automaton is a tuple `A=(Q, Œ£, Œ¥, initial_state, F)` where

- `Q` is a finite set of states
- `Œ£` is a finite set called the alphabet of `A`, i.e. `Œ£ = 2^AP`
- `Œ¥: Q √ó Œ£ ‚Üí Q` is the transition function
- `initial_state ‚àà Q` is the initial state
- `F ‚äÜ Q` is the set of accepting states

# Implementation

[environment_ltl_2d.hpp](../include/erl_env/environment_ltl_2d.hpp) defines the interface of LTL 2D environment. It
wraps [finite_state_automaton.hpp](../include/erl_env/finite_state_automaton.hpp), which accepts a YAML file as input,
which contains the following fields:

- `num_states`: number of states of the B√ºchi automaton, i.e. `Q`
- `atomic_propositions`: list of propositional variables, i.e. `AP`. Then `Œ£ = 2^AP` is set as `0, 1, ..., 2^|AP|-1`.
  Here we call each element of `Œ£` an edge label.
- `initial_state`: initial state of the B√ºchi automaton, i.e. `initial_state`
- `accepting_states`: list of accepting states of the B√ºchi automaton, i.e. `F`
- `transitions`: list of transitions of the B√ºchi automaton, i.e. `Œ¥`. Each transition is a tuple
  `((state, next_state), list of edge labels)`. The edge label is a w-word of the alphabet `Œ£`,
  i.e. a `|AP|-bit` binary number.
